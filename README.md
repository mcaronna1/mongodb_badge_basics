## One or two key differences that stood out to you between relational databases (e.g., MySQL) and non-relational/document databases
MySQL uses fixed tables, while MongoDB stores flexible, document-based data. MySQL usually grows by upgrading hardware, but MongoDB can grow across many servers. In healthcare, MongoDB works well for patient records because it can handle different types of data like lab resulst,and notes
## One healthcare-specific scenario where using a non-SQL database like MongoDB would be appropriate, and why?
MongoDB could be ideal for storing electronic health records because patient data such as doctors notes. Its flexible schema makes it easy to add new types of health data without redesigning the database.
